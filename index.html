Inspection Object

What we inspect.

The inspected object is a bare printed circuit board — its visual signature in shape, color and texture is what makes the four defect types separable.

01 · Shape

Flat rectangular board with periodic arrays of drilled vias, straight and curved copper traces, and rectangular IC footprints.

02 · Color

Green solder-mask substrate with silver-grey metallic pads and traces (HASL finish), plus occasional white silkscreen.

03 · Texture

Matte solder mask against specular metal pads in repetitive hole/pad patterns. Defects appear as local breaks in this regularity.

/ 02
Detection Method

Imaging system architecture.

Proposed AOI station: a controlled-lighting enclosure that captures a consistent image of every board, feeding the YOLOv12 detector.

Station layout

Side view

Field of View

FOV

Target board ≈ 100 × 100 mm; FOV ≈ 110 × 110 mm with margin. A 5 MP sensor yields ≈ 42 µm/pixel — fine enough to resolve sub-millimetre mouse-bites.

Lens · Camera · Lighting

Optics
  • Telecentric lens for near-zero perspective distortion on a flat board.
  • Global-shutter 5 MP CMOS to freeze conveyor motion.
  • Coaxial + low-angle ring reveal both flat copper and edge defects.

Detection Algorithm

Model

Single-stage YOLOv12 localises and classifies all four defect types in one forward pass; NMS + confidence threshold filter the boxes.

Mechanical Design

Rig

An anti-vibration base prevents motion blur; a triggered conveyor synchronises capture so each board is imaged at the same position.

/ 03
Detection Flow

From board to verdict.

01Capture

Image Capture

Industrial camera grabs the board when the trigger sensor fires.

02Preprocess

Resize · Normalize

Resize to 512×512 and normalise pixel values for the network.

03Infer

YOLOv12

One GPU forward pass (~15 ms) predicts defect boxes + classes.

04Filter

NMS · Threshold

Non-Max Suppression removes duplicates; confidence filter prunes weak boxes.

05Decide

Verdict + Log

Any defect ⇒ FAIL. Result logged; ≥3 consecutive fails raises SPC alarm.

code/dashboard.py · inspect_one() 
# ── INPUT ─────────────────────────────────────────────
img_path = image_paths[idx]              # one 512×512 PCB image

# ── PROCESS (YOLOv12 forward pass) ────────────────────
res = model.predict(
        source=img_path,
        conf=conf_threshold,             # confidence filter
        verbose=False,
)[0]                                     # NMS applied internally

names   = res.names
classes = [names[int(c)] for c in res.boxes.cls]   # class per box
confs   = res.boxes.conf.tolist()                  # confidence per box
annotated = res.plot()[:, :, ::-1]                 # draw boxes, BGR→RGB

# ── OUTPUT (decision + annotated image) ───────────────
verdict = "FAIL" if classes else "PASS"   # any defect ⇒ FAIL
return {"verdict": verdict, "classes": classes,
        "image": annotated, "confidences": confs}

Every result is appended to outputs/inspection_log.csv for traceability.

/ 06
Applications

Where else this transfers.

The same capture-then-detect recipe — controlled lighting, fixed optics, a fine-tuned detector — generalises to many surface-inspection problems.

Electronics

SMT solder joints

Detect bridging, insufficient solder, tombstoning on populated boards.

Semiconductor

Wafer & die inspection

Locate scratches, particles and pattern defects on wafers and dies.

Display

LCD / OLED panels

Find dead pixels, mura and line defects on display panels.

Hardware

Connectors & FPC

Inspect bent pins and flexible-PCB trace damage at high throughput.

General QC

Surface inspection

Metal, textile, glass or food surface defects with the same workflow.

Extension

More PCB classes

Add spur, spurious copper, or silkscreen OCR with more labelled data.

Course Knowledge

Mapping to Machine Vision Inspection.

Illumination

Choosing coaxial vs. low-angle ring vs. diffuse light to reveal defects — directly applies the lecture material on lighting geometry and reflection.

Optics — FOV, lens, resolution

Sizing the field of view, picking a telecentric lens, and computing µm/pixel spatial resolution come from the optics units.

Camera & acquisition

Sensor resolution, global vs. rolling shutter, and triggered capture relate to camera-selection and image-acquisition topics.

Image processing & detection

Preprocessing (resize / normalise) and the move from classical features to a deep-learning detector extend the image-processing lectures.

Inspection metrics

Precision, recall, mAP and the confusion matrix are the evaluation tools taught for judging an inspection system.

System integration & SPC

Packaging it as an AOI station with yield tracking and a consecutive-fail alarm applies the automation / SPC concepts.

/ 08
Reflection & Contribution

What I built and learned.

Reflection

Building this end to end made the course concepts concrete: data quality and augmentation matter as much as the model — brightness, contrast, flip and rotation are what let the detector hold up under realistic lighting and orientation changes. Working through the YOLOv12 training loop and reading the P / R / mAP curves taught me how to diagnose a model rather than just trust a single number. Wrapping everything in a simulated AOI dashboard showed me the gap between a model that scores well and an inspection system you could actually run — throughput, PASS/FAIL logic and operator-facing alarms all matter.
