Overview

An AI inspector for the production line

Manual visual inspection of printed circuit boards is slow, inconsistent, and fatiguing. This project trains a YOLOv12 detector on an augmented PCB dataset and wraps it in a simulated AOI station: test boards are streamed through the model one by one, producing live PASS / FAIL verdicts, a defect-class distribution, a yield trend, and an SPC-style alarm that fires after three consecutive failures — the same signals a real inline inspection cell would surface.

02 — Inspection Object

What we inspect, and how it looks

The inspected object is a bare printed circuit board (PCB). Understanding its visual signature — shape, color, texture — is what makes the four defect types separable.

Shape

Flat rectangular board with mounting holes at the corners; dense, periodic arrays of drilled holes/vias, straight & curved copper traces, and rectangular IC footprints.

Color

Green solder-mask substrate with silver-grey metallic pads, traces and hole rings (HASL finish), plus occasional white silkscreen.

Texture

Matte solder mask against specular, glossy metal pads; highly repetitive hole/pad patterns. Defects appear as local breaks in this regularity.

🔭 Field of View (FOV)

Target board ≈ 100 × 100 mm; FOV set to ≈ 110 × 110 mm with margin. With a 5 MP sensor (≈ 2592 px wide) this yields ≈ 42 µm/pixel — fine enough to resolve sub-millimetre mouse-bites and trace breaks.

📷 Lens · Camera · Lighting
  • Lens — telecentric, near-zero perspective distortion & constant magnification for accurate measurement on a flat board.
  • Camera — 5 MP CMOS area-scan, global shutter to freeze conveyor motion.
  • Lighting — coaxial light for uniform reflection off copper/pads, plus low-angle ring light that casts shadows to emphasise edge defects (mouse bite / open circuit).
🧠 Image-Processing Technique

A single-stage YOLOv12 object detector localises and classifies all four defect types in one forward pass; non-max suppression (NMS) and a confidence threshold filter the boxes.

⚙️ Mechanical Design

An anti-vibration base (damped mounts) prevents motion blur; a conveyor with a trigger sensor synchronises capture so each board is imaged at the same position.

04 — Detection Flow

From board to verdict

① Image Capture
Industrial camera grabs a board image when the trigger sensor fires.
→
② Pre-process
Resize to 512×512 and normalise pixel values for the network.
→
③ YOLOv12 Inference
One GPU forward pass (~15 ms) predicts defect boxes + classes.
→
④ Post-process
NMS removes duplicates; a confidence threshold filters weak boxes.
→
⑤ Verdict + Log
Any defect ⇒ FAIL. Result is logged; ≥3 fails raises an SPC alarm.

# ── INPUT ─────────────────────────────────────────────
img_path = image_paths[idx]              # one 512×512 PCB image

# ── PROCESS (YOLOv12 forward pass) ────────────────────
res = model.predict(
        source=img_path,
        conf=conf_threshold,             # confidence filter
        verbose=False,
)[0]                                     # NMS applied internally

names   = res.names
classes = [names[int(c)] for c in res.boxes.cls]   # class per box
confs   = res.boxes.conf.tolist()                  # confidence per box
annotated = res.plot()[:, :, ::-1]                 # draw boxes, BGR→RGB

# ── OUTPUT (decision + annotated image) ───────────────
verdict = "FAIL" if classes else "PASS"   # any defect ⇒ FAIL
return {"verdict": verdict, "classes": classes,
        "image": annotated, "confidences": confs}

The pipeline reads an image, runs a single YOLOv12 inference (built-in NMS + confidence filtering), then emits the annotated image and a PASS/FAIL verdict — every result is appended to outputs/inspection_log.csv.

07 — Applications

Where else this transfers

The same capture-then-detect recipe — controlled lighting, fixed optics, a fine-tuned detector — generalises to many surface-inspection problems.

🔧

SMT solder-joint inspection

Detect bridging, insufficient solder, tombstoning on populated boards.

💾

Semiconductor & wafer

Locate scratches, particles and pattern defects on wafers and dies.

📺

Display panels

Find dead pixels, mura and line defects on LCD/OLED panels.

🔌

Connectors & FPC

Inspect bent pins and flexible-PCB trace damage.

🏭

General surface QC

Metal, textile, glass or food surface defects with the same workflow.

➕

Extended PCB classes

Add spur, spurious copper, or silkscreen-OCR with more labelled data.

09 — Reflection & Contribution

What I built and learned

Reflection

Building this end to end made the course concepts concrete: I learned that data quality and augmentation matter as much as the model — brightness, contrast, flip and rotation were what let the detector hold up under realistic lighting and orientation changes. Working through the YOLOv12 training loop and reading the precision / recall / mAP curves taught me how to diagnose a model rather than just trust a single number. Wrapping everything in a simulated AOI dashboard showed me the gap between "a model that scores well" and "an inspection system you could actually run" — throughput, PASS/FAIL logic, and operator-facing alarms all matter.
